## 2) أوامر الإثبات الإلزامية

1. فحص الالتزام بعد تاريخ القطع:

```bash
npm run coa:cutover:check --workspace=backend
```

2. توليد حزمة إثبات UAT المالي:

```bash
npm run coa:uat:evidence --workspace=backend
```

3. مخرجات يجب إرفاقها مع النموذج:
- ملف JSON من `backend/reports/coa-financial-uat-evidence-*.json`
- ملف MD من `backend/reports/coa-financial-uat-evidence-*.md`
- ملف compliance من `backend/reports/coa-cutover-compliance-*.json`

## 3) قائمة التحقق المالية (Pass/Fail)

| # | بند التحقق | النتيجة | ملاحظات |
|---|---|---|---|
| 1 | لا يوجد ترحيل على Header بعد تاريخ القطع | ☐ Pass ☐ Fail | |
| 2 | جميع `gl_account_defaults` مرتبطة بحسابات Posting نشطة | ☐ Pass ☐ Fail | |
| 3 | Trial Balance متوازن (Debit = Credit) | ☐ Pass ☐ Fail | |
| 4 | Balance Sheet متوازن (Assets = Liabilities + Equity) | ☐ Pass ☐ Fail | |
